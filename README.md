# 代码比对报告工具

Windows 桌面应用，输入 Git/SVN/文件夹/压缩包 项目路径和两个版本号，一键生成 HTML 差异报告并导出变更文件。通过 PyInstaller 打包成单文件 exe，无需 Python 环境。

## 功能

- 支持 **Git**、**SVN**、**文件夹对比**和**压缩包比对**四种模式
- Git/SVN 模式一键获取版本列表（含提交注释），双击填入
- 压缩包模式支持 `.zip` / `.tar` / `.tar.gz` / `.tar.bz2`，自动处理中文文件名编码
- 输入新旧版本号，自动生成**左右分栏对比**的 HTML 报告
- 左侧文件树，按颜色标记：🟢 新增 / 🔵 修改 / 🔴 删除
- 文件树支持「全部 / 新增 / 修改 / 删除」过滤标签，可任意组合
- 支持目录折叠展开，一键全部展开/折叠
- 点击文件查看代码差异（新增/修改/删除行高亮，行号着色）
- **变更清单弹窗**，按新增/修改/删除分类列出所有变更文件
- 差异展示方式可选：**全部内容** 或 **仅差异上下文**（前后各 3 行）
- 自动导出新版本和旧版本的变更文件，空文件正常导出
- 导出文件自动匹配 Windows 换行符（Git autocrlf / SVN eol-style），与工作副本字节一致
- 支持 **glob 排除规则**（`*.class`、`target/**` 等），避免编译产物干扰
- 二进制/归档文件（`.jar`、`.war`、`.class` 等）标记变更但不展示内容
- SVN 可执行文件路径自动探测（PATH → 注册表 → 常见安装目录），亦可手动指定
- 中文编码自动适配（子进程输出 + 文件内容均 UTF-8 → GBK 回退）
- 配置自动记忆，下次打开恢复上次设置

## 运行

### 开发环境

```bash
pip install jinja2
python main.py
```

### 打包为 exe

```bash
build.bat
```

打包后的 exe 在 `dist/CompareTool.exe`。使用 `--console` 模式打包，确保 git/svn 子进程有终端可用。

## 使用说明

1. 选择项目目录，选择版本控制类型（Git / SVN / 文件夹 / 压缩包）
2. Git/SVN 模式点击「获取版本列表」查看可用版本（含提交注释，单击选中后点「←填入」或双击直接填入）
3. 输入/修改排除规则（支持 `*` 和 `**` 通配符）
4. 设置输出路径，点击「生成比对报告」
5. 生成完毕后自动询问是否打开报告

**文件夹对比模式**：选择两个文件夹分别作为旧版本和新版本，直接比对文件差异（基于 `filecmp.cmp` 内容比较），无需 VCS。

**压缩包比对模式**：选择两个压缩包作为旧版本和新版本，工具自动解压到临时目录后比对。支持 `.zip`、`.tar`、`.tar.gz`、`.tar.bz2` 格式。

## 排除规则

项目目录下的 `paichu.txt` 定义默认排除规则，每行一个 glob 模式：

```
*.class
target/**
.git/**
.idea/**
node_modules/**
...
```

规则语法：
- `*.class` — 匹配任意深度的 .class 文件
- `target/**` — 匹配 target 目录下所有文件
- `**/test/**` — 匹配任意位置的 test 目录

## 配置文件

`compareTool_config.json` 自动保存上次的项目路径、VCS 类型、SVN 路径、排除规则、输出路径。排除规则优先级：用户保存的配置 > `paichu.txt` > 内置默认值。
